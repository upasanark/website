Chapter 5

Graph Transformers

Beyond message passing: self-attention on graphs, positional encodings, Graphormer, hybrid architectures, and efficient scaling

Limitations of Message Passing

1. Fixed receptive field: K-layer GNN sees only K-hop neighbors
2. Over-squashing: information from distant nodes compressed through bottleneck

\(\frac{\partial h_v^{(K)}}{\partial x_u}\) decays exponentially with distance\((u,v)\)

3. Structural blindness: GNNs bounded by WL test — can't distinguish certain non-isomorphic graphs
What if a node could attend to ANY other node, regardless of graph distance? That's the motivation for Graph Transformers.

Self-Attention on Graphs

Standard Transformer Attention:

\[ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V \]

On graphs: \(Q = H W_Q\), \(K = H W_K\), \(V = H W_V\) where \(H \in \mathbb{R}^{n \times d}\) = all node features

Result: every node can "see" every other node — global receptive field in one layer
Cost: \(O(n^2 \cdot d)\) — quadratic in number of nodes
Problem: ignores graph structure entirely (treats graph as complete graph)
Solution: incorporate structure via positional encodings + attention bias terms
A naive graph transformer throws away the adjacency matrix — defeating the purpose. The art is encoding structure INTO the attention mechanism so the model knows who is a neighbor and who is distant.

Positional Encodings for Graphs

Problem: graphs have no inherent node order (unlike sequences with position 1, 2, 3…)

1. Laplacian Eigenvector PE

\[ \text{PE}(v) = [\phi_1(v),\; \phi_2(v),\; \ldots,\; \phi_k(v)] \]

First \(k\) eigenvectors of graph Laplacian \(L = D - A\), evaluated at node \(v\)

⚠️ Sign ambiguity: \(\phi\) and \(-\phi\) are both valid eigenvectors

2. Random Walk PE (RWPE)

\[ \text{RWPE}(v) = [RW^1_{vv},\; RW^2_{vv},\; \ldots,\; RW^K_{vv}] \]

Landing probability of returning to \(v\) after \(k\) steps — captures local structure (triangles, cycles)

✓ No sign ambiguity, computationally cheap \(O(K \cdot |E|)\)

3. Learned PE

Train a small GNN to generate positional features — adapts to task

Eigenvectors tell you "where" a node sits in the global structure (center vs periphery, bridge vs cluster). Random walk probabilities tell you about local topology. Both inform the transformer about graph geometry.

Graphormer (Ying et al., 2021)

Three structural encodings injected into a standard Transformer:

1. Centrality Encoding: node degree as bias to input embeddings
\(\quad h_v^{(0)} = x_v + b_{\text{deg}^-(v)} + b_{\text{deg}^+(v)}\)
2. Spatial Encoding: shortest-path distance as attention bias
\(\quad A_{ij} \mathrel{+}= b_{\text{SPD}}(d(v_i, v_j))\) — closer nodes get attention boost
3. Edge Encoding: aggregate edge features along shortest path
\(\quad E_{ij} = \frac{1}{|SP(i,j)|} \sum_{e \in SP(i,j)} x_e \cdot w_e\)

Modified Attention:

\[ A_{ij} = \frac{(h_i W_Q)(h_j W_K)^T}{\sqrt{d}} + b_{\text{spatial}}(i,j) + b_{\text{edge}}(i,j) \]

Results: SOTA on OGB molecular benchmarks (OGB-LSC)

Limitation: all-pairs shortest paths = \(O(n^2)\) precomputation

Graphormer doesn't replace graph structure — it encodes it INTO the attention mechanism. The graph topology becomes soft biases that guide where attention flows.

Hybrid Approaches — Local + Global

Strategy: use GNN for local aggregation, Transformer for global context

Architecture Patterns:

1. GNN → Transformer (Sequential): GNN embeddings fed into transformer layers
2. Parallel: GNN branch + Transformer branch → combine outputs
3. Interleaved: alternate GNN and Transformer layers

GPS Framework (Rampášek et al., 2022):

\[ h_v^{(l+1)} = h_v^{(l)} + \underbrace{\text{GNN}_{\text{local}}(h_v^{(l)}, \{h_u^{(l)}: u \in \mathcal{N}(v)\})}_{\text{local neighborhood}} + \underbrace{\text{Transformer}_{\text{global}}(h_v^{(l)}, H^{(l)})}_{\text{global attention}} \]

Each layer: local captures neighborhood structure, global captures long-range dependencies

PatternLocal?Global?Trade-off
Pure GNN✓✗Fast, limited range
Pure Transformer✗✓No structure bias
GPS (Hybrid)✓✓More params, best expressiveness
The best of both worlds: GNN handles neighborhood patterns (cheap, structured), Transformer handles long-range dependencies (expensive, flexible).

Efficient Graph Transformers

Full attention: \(O(n^2)\) — impractical for million-node graphs

1. Sparse Attention: attend only to graph neighbors + sampled distant nodes → \(O(n \cdot k)\)
2. Performer / Linear Attention: approximate softmax with random features
\(\quad \text{softmax}(QK^T) \approx \phi(Q)\phi(K)^T\) → \(O(n \cdot d^2)\) linear in nodes
3. Cluster-Based: cluster nodes → attend within clusters + across cluster representatives
4. Sampling: attend to random subset of nodes each layer (like dropout for attention)

For real recommendation graphs (millions of nodes), you MUST use sparse or linear attention. The question becomes: which nodes deserve attention?

Graph Transformers for Recommendations

Why transformers for RecSys graphs?

Long-range dependencies: user A → item X → user B → item Y (4-hop collaborative path)
GNN needs 4 layers (over-smoothing risk) — Transformer attends directly

Sequential Recommendation as Graph Transformers:

SASRec: left-to-right self-attention on interaction sequence

BERT4Rec: bidirectional masked-item prediction

Both are "sequence transformers on the interaction graph" — treating temporal item sequence as a directed path

Cross-Attention between Users and Items:

User representations attend to item set; items attend to user set → learned bipartite interactions

Current State: graph transformers for RecSys are emerging but not yet dominant
• Compute cost vs marginal gain over simpler GNN methods (LightGCN)
• Best results: hybrid architectures (GNN backbone + transformer refinement)
For recommendations, the transformer's strength is capturing non-local collaborative signals that a 2-layer GNN would miss. But the cost-benefit trade-off is still being figured out.

Quiz — Chapter 5

Q1: What is over-squashing and why does it limit deep GNNs?
A: Over-squashing occurs when information from exponentially many distant nodes must be compressed into a fixed-size vector at intermediate nodes. Mathematically, \(\partial h_v^{(K)} / \partial x_u\) decays exponentially with \(d(u,v)\), meaning distant signals vanish. This limits deep GNNs because stacking more layers doesn't help — the gradient signal from far nodes is destroyed.
Q2: Laplacian eigenvector PEs have sign ambiguity. Why is this a problem for learning?
A: If \(\phi\) is an eigenvector, so is \(-\phi\). Two isomorphic substructures could receive opposite PE signs, making the model's learned function inconsistent. During training, the same structural role maps to different PE values across graphs, preventing stable learning. Solutions: use \(|\phi|\), SignNet, or switch to Random Walk PE (which has no ambiguity since probabilities are always positive).
Q3: In Graphormer, spatial encoding adds \(b_{\text{SPD}}(d(i,j))\) to attention. If nodes \(i,j\) are 5 hops apart and the learned bias table only has entries up to distance 3, what happens?
A: Distances beyond the threshold are typically clipped or mapped to a single "far" bucket. So \(b_{\text{SPD}}(5)\) would use the same learned bias as all pairs with distance ≥ 3 (or whatever the cap is). This means the model treats all "distant enough" node pairs equally — losing fine-grained long-range distinctions.

Quiz — Chapter 5 (continued)

Q4: Why might a hybrid GNN+Transformer outperform a pure Graph Transformer on recommendation graphs?
A: (1) The GNN component efficiently captures strong local collaborative signals (direct co-purchase, shared neighbors) which are the dominant signal in RecSys. (2) The Transformer adds long-range dependencies without requiring deep GNN stacks (avoids over-smoothing). (3) A pure Transformer with no structural bias wastes capacity re-learning locality from data. The hybrid gets locality "for free" from the GNN and uses transformer capacity for genuinely non-local patterns.
Q5: For a graph with 10M nodes, full self-attention requires \(O(10^{14})\) operations. Name two strategies to make this tractable.
A: (1) Sparse attention — only attend to graph neighbors + sampled distant nodes, reducing to \(O(n \cdot k)\) where \(k \ll n\). (2) Linear attention (Performer) — approximate softmax with random feature maps: \(\text{softmax}(QK^T) \approx \phi(Q)\phi(K)^T\), reducing to \(O(n \cdot d^2)\). Other valid answers: cluster-based attention, node sampling, or restricting attention to local subgraphs.
