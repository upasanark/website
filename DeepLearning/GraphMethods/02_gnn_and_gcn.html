Chapter 2

Graph Neural Networks & Graph Convolution

Message passing, spectral convolutions, GCN, GraphSAGE, GAT, and the over-smoothing problem

The Core Idea — Learning on Graph Structure

Problem:

We have a graph \(G\) + node features \(X \in \mathbb{R}^{n \times d}\)

Want to learn node representations \(H \in \mathbb{R}^{n \times d'}\) that capture both features AND structure

Solution:

Let each node's representation depend on its neighborhood

If your friends all like jazz, you probably like jazz too — that's GNN in one sentence.

Message Passing Framework

Three phases per layer:

1. Message: \(\quad m_{j \to i}^{(l)} = \text{MSG}(h_i^{(l)}, h_j^{(l)}, e_{ij})\)

2. Aggregate: \(\quad M_i^{(l)} = \text{AGG}\left(\{m_{j \to i}^{(l)} : j \in \mathcal{N}(i)\}\right)\)

3. Update: \(\quad h_i^{(l+1)} = \text{UPDATE}(h_i^{(l)}, M_i^{(l)})\)

Each node collects information from neighbors (messages), combines them (aggregate), and updates itself. This IS the GNN.

Aggregation Functions

Mean: \(\quad M_i = \frac{1}{|\mathcal{N}(i)|} \sum_{j \in \mathcal{N}(i)} h_j\) — averages neighbor features

Sum: \(\quad M_i = \sum_{j \in \mathcal{N}(i)} h_j\) — preserves degree information

Max: \(\quad M_i = \max_{j \in \mathcal{N}(i)} h_j\) — captures most salient neighbor

AggregatorDegree-aware?ProsCons
MeanNo (normalizes out)Stable, smoothLoses count info
SumYesDistinguishes substructuresSensitive to degree
MaxNoRobust to noiseLoses most info
Mean: "What's the average opinion of my friends?" Sum: "What's the TOTAL signal from my neighborhood?" Max: "What's the strongest single signal?"

Multi-Hop Propagation — Receptive Field

Layer 1: node sees 1-hop neighbors

Layer 2: node sees 2-hop (neighbors of neighbors)

Layer K: node sees K-hop neighborhood


\(h_i^{(K)}\) encodes information from all nodes within K hops

After 2 layers on a user-item graph:

User "sees" items purchased by similar users (2-hop collaborative signal)

Each GNN layer expands the receptive field by 1 hop. K layers = K-hop neighborhood. This is how collaborative filtering emerges from graph structure.

Spectral Graph Convolutions — The Theory

Goal: define convolution on graphs

On images: convolution = slide a filter over a regular grid. On graphs: no grid — nodes are irregularly connected.

Key tool: Graph Fourier Transform using Laplacian eigenvectors

Eigendecomposition: \(\; L = U \Lambda U^T\)


Graph Fourier Transform: \(\hat{x} = U^T x\)

Inverse: \(x = U \hat{x}\)


Spectral convolution:

\[ g_\theta \star x = U \, g_\theta(\Lambda) \, U^T x \]

where \(g_\theta(\Lambda)\) is a learnable filter in the spectral domain

Problem: computing eigenvectors is \(O(n^3)\) — not scalable!

Convolution on images = local pattern detection. On graphs, the same idea — but "locality" is defined by graph structure, not pixel adjacency. The spectral approach gives us the theory; we need approximations for practice.

Chebyshev Approximation — Making It Scalable

Approximate \(g_\theta(\Lambda)\) with Chebyshev polynomials up to order K:

\[ g_\theta(\Lambda) \approx \sum_{k=0}^{K} \theta_k \, T_k(\tilde{\Lambda}) \]

where \(\tilde{\Lambda} = \frac{2\Lambda}{\lambda_{max}} - I\) (scaled to [-1, 1])


Chebyshev recurrence:

\[ T_0(x) = 1, \quad T_1(x) = x, \quad T_k(x) = 2x \cdot T_{k-1}(x) - T_{k-2}(x) \]

Why this works:

  • \(T_k(L)\) only requires K-hop neighborhoods (localized!)
  • No eigenvector computation needed — just multiply by \(L\) repeatedly
  • K parameters to learn (not n), computation is \(O(K \cdot |E|)\)
Chebyshev polynomials let us approximate any spectral filter using only local graph operations. Order K = K-hop locality. This bridges spectral theory to spatial implementation.

GCN — The Key Simplification (Kipf & Welling, 2017)

Set K=1, assume λmax≈2 in Chebyshev → first-order approximation

Result: \(\; g_\theta \star x \approx \theta \left( I + D^{-1/2} A D^{-1/2} \right) x\)


Renormalization trick: replace \(I + D^{-1/2}AD^{-1/2}\) with \(\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}\)

where \(\tilde{A} = A + I\) (add self-loops) and \(\tilde{D}_{ii} = \sum_j \tilde{A}_{ij}\)


The GCN Layer:

\[ H^{(l+1)} = \sigma\!\left( \tilde{D}^{-1/2} \tilde{A} \, \tilde{D}^{-1/2} \, H^{(l)} W^{(l)} \right) \]
  • \(\tilde{D}^{-1/2} \tilde{A} \, \tilde{D}^{-1/2}\): normalized aggregation (including self)
  • \(W^{(l)}\): learnable weight matrix (feature transformation)
  • \(\sigma\): nonlinearity (ReLU)
GCN took the elegant but expensive spectral theory and boiled it down to: "average your neighbors' features (including yourself), then transform." One line of code.

GCN Layer — Dissecting Each Component

\[ H^{(l+1)} = \sigma\!\left( \underbrace{\tilde{D}^{-1/2} \tilde{A} \, \tilde{D}^{-1/2}}_{\text{normalized aggregation}} \underbrace{H^{(l)}}_{\text{current features}} \underbrace{W^{(l)}}_{\text{transformation}} \right) \]
ComponentWhat it doesWithout it
\(\tilde{A} = A + I\)Includes self-loop (node keeps its own features)Node forgets itself after aggregation
\(\tilde{D}^{-1/2} \cdot \tilde{D}^{-1/2}\)Symmetric normalization: scale by \(1/\sqrt{d_i \cdot d_j}\)High-degree nodes dominate
\(W^{(l)}\)Learnable projection (d → d')Can't change feature dimension
\(\sigma\) (ReLU)Nonlinearity between layersMultiple layers collapse to one
The normalization ensures that a node with 10,000 neighbors doesn't overwhelm a node with 5 neighbors. Every neighbor gets "voting power" proportional to 1/√(their degree × your degree).

GCN — Worked Example

4-node graph, 2-dim features, 1 GCN layer (no W, no σ for clarity):

Features:

\(x_1 = [1, 0]\), \(x_2 = [0, 1]\)

\(x_3 = [1, 1]\), \(x_4 = [0, 0]\)

Edges: (1,2), (1,3), (2,3), (3,4)

Step 1: Add self-loops → \(\tilde{A} = A + I\)

Step 2: Compute \(\tilde{D}\): degrees [3,3,4,2]

Step 3: \(\hat{A} = \tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}\)

Node 1's new embedding (aggregating from neighbors 2, 3, and self):

\[ h_1' = \frac{1}{\sqrt{3}} \left( \frac{x_1}{\sqrt{3}} + \frac{x_2}{\sqrt{3}} + \frac{x_3}{\sqrt{4}} \right) = \frac{1}{3}[1,0] + \frac{1}{3}[0,1] + \frac{1}{\sqrt{12}}[1,1] \] \[ h_1' \approx [0.62, 0.62] \]

Node 1 started at [1,0] but after aggregation moves toward its neighbors' average!

After one GCN layer, nodes with similar neighborhoods get similar embeddings. This is how "guilt by association" becomes a mathematical operation.

GraphSAGE — Inductive Learning (Hamilton et al., 2017)

Problem with GCN:

Transductive — needs full graph at training time. Can't embed unseen nodes.

Solution: Sample + Aggregate

\[ h_v^{(l)} = \sigma\!\left( W^{(l)} \cdot \text{CONCAT}\!\left( h_v^{(l-1)},\; \text{AGG}\!\left(\{h_u^{(l-1)} : u \in S(\mathcal{N}(v))\}\right) \right) \right) \]
Algorithm:
1. SAMPLE: pick K random neighbors (fixed budget)
2. AGGREGATE: mean/LSTM/pool over sampled neighbors
3. CONCAT: combine self-embedding with aggregation
4. TRANSFORM: linear projection + nonlinearity

Key advantage:

  • Can generalize to unseen nodes (new users/items)
  • Fixed computation budget per node (sample K neighbors)
  • Mini-batch training possible (no full graph in memory)
GraphSAGE learns a FUNCTION for aggregating neighborhoods — not fixed embeddings. New nodes just run the function on their neighbors.

GAT — Graph Attention Networks (Veličković et al., 2018)

Motivation: not all neighbors are equally important

Attention coefficient:

\[ e_{ij} = \text{LeakyReLU}\!\left( \mathbf{a}^T [W h_i \,\|\, W h_j] \right) \]

Normalized (softmax over neighbors):

\[ \alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{k \in \mathcal{N}(i)} \exp(e_{ik})} \]

Aggregation:

\[ h_i' = \sigma\!\left( \sum_{j \in \mathcal{N}(i)} \alpha_{ij} \cdot W h_j \right) \]

Multi-head attention: K heads, concatenate or average outputs

MethodNeighbor weighting
GCNFixed: \(1/\sqrt{d_i \cdot d_j}\) (from degree only)
GATLearned: \(\alpha_{ij}\) (from content of both nodes)
If GCN treats all neighbors equally, GAT asks "which neighbors matter most for THIS node?" — and learns the answer from data.

The Over-Smoothing Problem

Observation: As layers increase, all node embeddings converge to the same vector.

\[ \lim_{K \to \infty} h_i^{(K)} \to \frac{d_i}{2m} \cdot \mathbf{1} \quad \text{(proportional to degree)} \]

Why it happens:

  • Repeated mean aggregation = low-pass filter
  • Each layer smooths differences between connected nodes
  • After many layers: all high-frequency (discriminative) signal is killed
  • Analogy: repeatedly averaging pixel colors → entire image becomes grey

Practical consequence:

LayersReceptive fieldQuality
1-2LocalGood
3BroaderMarginal gain
4+Near-globalDegrades (over-smoothing)
Over-smoothing = every node looks the same after too many layers. This is why GNNs are shallow (2-3 layers) — deeper ≠ better on graphs.

Mitigating Over-Smoothing

1. Residual Connections

\[ h_i^{(l+1)} = h_i^{(l)} + \text{GNN\_layer}(h_i^{(l)}, \ldots) \]

Preserve original features while adding neighborhood info

2. Jumping Knowledge (JK-Net)

\[ h_i^{\text{final}} = f\!\left(h_i^{(0)}, h_i^{(1)}, \ldots, h_i^{(K)}\right) \]

Combine ALL layer outputs: concat, max-pool, or attention over layers

3. DropEdge

Randomly remove edges during training → prevents message over-propagation

4. APPNP (Personalized PageRank)

\[ H^{(k)} = \alpha \cdot \hat{A} \cdot H^{(k-1)} + (1-\alpha) \cdot H^{(0)} \]

Always "teleport" back to initial features with probability \((1-\alpha)\)

The fix: don't let the GNN forget the original signal. Skip connections, layer combination, and teleportation all keep the initial features alive.

GNN Methods — Comparison

Unifying view — all are: \(\quad h_i^{(l+1)} = \sum_{j \in \mathcal{N}(i) \cup \{i\}} c_{ij} \cdot \text{TRANSFORM}(h_j^{(l)})\)

They differ only in how \(c_{ij}\) is computed and whether TRANSFORM is learned.

Quiz — Chapter 2

Q1: In the message passing framework, what are the three phases?
A: Message (compute per-edge messages), Aggregate (combine incoming messages), Update (produce new node embedding)
Q2: GCN adds self-loops (A+I). What would happen without them?
A: Without self-loops, a node's own features are NOT included in aggregation. After one layer, the node's embedding is purely a function of its neighbors — it "forgets" itself entirely.
Q3: If a node has 3 neighbors with embeddings [1,0], [0,1], [1,1], what is the mean aggregation result?
A: Mean = ([1,0] + [0,1] + [1,1]) / 3 = [2/3, 2/3] ≈ [0.67, 0.67]

Quiz — Chapter 2 (continued)

Q4: Why does GraphSAGE sample neighborhoods rather than using all neighbors?
A: (1) Fixed computation budget per node — scalable to any graph size. (2) Enables mini-batch training — don't need full graph in memory. (3) Makes the model inductive — same function works on any neighborhood.
Q5: In GAT, attention coefficients αij sum to 1 over j∈N(i). What function ensures this?
A: Softmax over the neighborhood: αij = exp(eij) / Σk∈N(i) exp(eik). This normalizes attention weights to a valid probability distribution.
Q6: True/False: A 10-layer GCN will always outperform a 2-layer GCN because it captures higher-order structure.
A: False. Over-smoothing causes all embeddings to converge after ~4 layers. A 10-layer GCN will perform WORSE than 2-3 layers on most tasks. Deeper ≠ better on graphs.
