Chapter 3

Item Tokenization: The Most Important Design Decision

RQ-VAE, OPQ, semantic hierarchies, hybrid approaches, and the collision ceiling

Why Tokenization is the Ceiling

Core Theorem (informal):

Generation accuracy ≤ tokenization quality

  • If two items share the same token sequence (collision) → model CANNOT distinguish them
  • Collision rate directly bounds achievable Recall@1
  • All architecture improvements are meaningless if tokens are bad
  • Token quality > architecture quality

Bound:

\[\text{max Recall@1} \leq \frac{|\text{unique token sequences}|}{|\text{total items}|}\]

"You can build the best language model in the world, but if your vocabulary is ambiguous, it will always hallucinate."

RQ-VAE — Residual Quantization (TIGER, 2023)

Input: item embedding \(e \in \mathbb{R}^d\) from content model or CF

Step 1: Find nearest centroid: \(c_1 = \arg\min_k \|e - \text{codebook}_1[k]\|\)
Step 2: Compute residual: \(r_1 = e - \text{codebook}_1[c_1]\)
Step 3: Quantize residual: \(c_2 = \arg\min_k \|r_1 - \text{codebook}_2[k]\|\)
Step 4: Repeat for \(K\) levels → Output: \([c_1, c_2, \ldots, c_K]\)

Reconstruction: \(\quad e \approx \sum_{k=1}^{K} \text{codebook}_k[c_k]\)

Commitment Loss: \(\quad \mathcal{L} = \|e - \text{sg}(\hat{e})\|^2 + \beta\|\text{sg}(e) - \hat{e}\|^2\)

Gradient: Straight-through estimator (copy gradients past argmin)

RQ-VAE — Properties & Limitations

Properties ✓

  • Hierarchical: Level 1 = coarse cluster, Level K = fine discrimination
  • Semantic prefix: similar items → similar token prefixes (soft property)
  • Uniqueness optimized: training minimizes collisions
  • End-to-end: can train jointly with downstream task

Limitations ✗

  • Opaque codes: token [3, 17, 42, 8] means nothing to humans
  • Cold-start: new items need encoder pass through RQ-VAE
  • Can't constrain: "only generate electronics" is impossible (codes are opaque)
  • Codebook collapse: some centroids go unused → wasted capacity

Typical config: codebook size = 256–1024 per level, K = 4–8 levels

OPQ — Optimized Product Quantization

Different from RQ-VAE: splits dimensions (not residuals)

Step 1: Learn rotation matrix \(R\) to decorrelate dimensions
Step 2: Split rotated embedding into \(M\) subvectors: \(\hat{e} = R \cdot e \rightarrow [\hat{e}_1, \ldots, \hat{e}_M]\)
Step 3: Quantize each subvector independently (\(K\) centroids per subspace)
Output: Item ID = \([c_1, c_2, \ldots, c_M]\) where each \(c_m \in \{1,\ldots,K\}\)

Key advantage: Each subspace captures a different "aspect" — one might capture price-range, another category, another brand style

Used for L7 spec tokens: OPQ on Titan Embed summaries → 4 spec tokens (256 codes each). These provide the discriminative power to break collisions in the semantic levels.

CoFiRec — Collaborative Filtering in Tokenization (Nov 2025)

Problem: Pure content tokens miss co-purchase/co-interaction signal

Stage 1: Content-based clustering (as in RQ-VAE)
Stage 2: Refine cluster assignments using CF signal (co-interaction graph)
Result: Items frequently co-purchased get similar token prefixes

Tokens now encode BOTH content similarity AND collaborative similarity

Key finding: +15–20% Recall improvement over content-only tokenization

"Tokenization quality IS the ceiling." CoFiRec proves that better tokens > better architecture. The CF signal from graph propagation directly improves token quality.

Semantic / Attribute-Based Tokenization

Use human-interpretable attributes as tokens directly

Simple Hierarchy:

Category → Subcategory → Brand → Price Tier

Rich Hierarchy (GenRec):

Family(L1) → Ecosystem(L3) → Brand(L4) → Price(L5) → Intent(L6) → Specs(L7)

LLM-classified:

Prompt LLM to classify items into categories/intents

Collision Handling:

Multiple items can share same attribute tuple

Solution 1: Add spec tokens (learned) to increase uniqueness
Solution 2: Item ID head for de-collision at inference

Hybrid Approaches

Dual Identifiers (SIGIR-AP 2025): Two decoder heads — one generates semantic tokens, one generates RQ-VAE codes. Score = combination of both generation probabilities.
KGTB: Knowledge graph entity path = token path. Items inherit tokens from their KG position (entity → parent → root).
GenRec (Your approach): L1–L6 semantic (interpretable, constrainable) + L7 learned OPQ (discriminative).
  • L1–L6: 18 + 4 + 50 + 4 + 18 = <100 codes total (human-readable)
  • L7a–d: OPQ on Titan Embed → 256 codes × 4 subspaces
  • Result: ~90% unique among 45K items

Collision Analysis — The Practical Ceiling

Definition: Collision = multiple items share the SAME complete token sequence

Impact: If 3 items collide → model can't distinguish → Recall@1 ≤ 1/3 for those items

Theoretical max: \(\text{Recall@1}_{\max} = \frac{|\text{unique items}|}{|\text{total items}|}\)

Reducing Collisions:

  • More token levels (deeper hierarchy)
  • Larger codebook per level
  • Better embedding input (more distinctive)
  • Domain-specific attributes
90% unique → max token-only R@1 ≈ 0.90

With ID head: can recover some collisions via discriminative scoring at inference.

Chapter 3 — Summary

Key Takeaways:

  1. Tokenization is the ceiling — collision rate directly bounds max Recall@1
  2. RQ-VAE quantizes residuals hierarchically; good uniqueness, but opaque
  3. OPQ splits dimensions after rotation; each subspace captures different aspects
  4. CoFiRec adds CF signal to tokenization → +15–20% Recall
  5. Semantic tokens are interpretable and constrainable but have more collisions

Design Principles:

  1. Hybrid > pure approach
  2. Coarse levels → semantic; fine levels → learned
  3. CF signal improves any tokenization
  4. Measure collision rate as primary quality metric
  5. More levels with small codebooks > fewer levels with big codebooks

Quiz — Item Tokenization

Q1: If 10% of items have collisions, what is the theoretical maximum Recall@1 from token generation alone?
A: ≈ 0.90. Max Recall@1 = unique_items / total_items. If 10% collide, only 90% can ever be correctly identified by tokens alone. The remaining 10% need a disambiguation mechanism (ID head).
Q2: What is the key difference between RQ-VAE and OPQ in how they decompose embeddings?
A: RQ-VAE quantizes residuals (error from previous level) sequentially. OPQ rotates and splits dimensions into subvectors, quantizing each independently. RQ-VAE is hierarchical depth; OPQ is parallel subspaces.
Q3: How does CoFiRec improve over standard content-based tokenization?
A: CoFiRec refines content-based cluster assignments using collaborative filtering signal (co-interaction graph). Items that are frequently co-purchased get similar token prefixes, encoding both content AND collaborative similarity. This yields +15–20% Recall improvement.

Quiz — Item Tokenization (continued)

Q4: In the GenRec hybrid approach, why is L7 learned (OPQ) instead of semantic like L1–L6?
A: L1–L6 semantic levels provide interpretability and constraint support, but they can't uniquely identify items (too many share the same attributes). L7 learned tokens (OPQ on Titan Embed) add discriminative power, breaking collisions by capturing fine-grained item differences that don't map to clean human attributes. Result: 90% uniqueness vs ~60% with semantic alone.
Q5: Why do Dual Identifier systems use two separate decoder heads instead of one combined token sequence?
A: Semantic tokens and collaborative tokens live in different spaces with different vocabularies and different learned distributions. A single sequence would force the model to context-switch between two incompatible token types. Two heads let each specialize: semantic head provides interpretable constraints; collaborative head provides fine-grained ranking. The combined score fuses both signals.
